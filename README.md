# Lab-5-


Вербальна модель предметної області
Предметна область — система управління продажами торгового підприємства ТОВ «SalesPro», що спеціалізується на оптовому та роздрібному продажу товарів. Система охоплює повний ланцюг від отримання замовлення до отримання оплати.

Основні сутності та їх призначення:
–	Клієнти — юридичні та фізичні особи, що здійснюють замовлення. Мають кредитний ліміт та контактні дані;
–	Менеджери — співробітники відділу продажів, що обслуговують клієнтів. Мають ієрархію підпорядкування;
–	Замовлення — факт оформлення покупки, пов'язаний з клієнтом та менеджером;
–	Товари — номенклатурні одиниці з цінами та ієрархічними категоріями;
–	Постачальники — зовнішні партнери, що постачають товар;
–	Склади — місця зберігання товару із залишками та резервами;
–	Відвантаження та доставки — логістичні операції передачі товару клієнту;
–	Рахунки та оплати — фінансові документи та факти надходження коштів.

Ітерація 3 — Зв'язки (ідентифікуючі та неідентифікуючі)

<img width="663" height="430" alt="image" src="https://github.com/user-attachments/assets/21690174-d194-418b-b3b0-d5677a35f9c1" />

Між сутностями встановлено зв'язки двох типів:
–	Ідентифікуючий зв'язок (solid line) — дочірня сутність не може існувати без батьківської. PK дочірньої включає FK батьківської. Приклад: Рядок_Замовлення ідентифікується через Замовлення;
–	Неідентифікуючий зв'язок (dashed line) — дочірня сутність може існувати незалежно. FK не входить до PK. Приклад: Клієнт → Замовлення (клієнт існує без замовлень).

Ітерація 5 — Рекурсивні зв'язки

<img width="652" height="348" alt="image" src="https://github.com/user-attachments/assets/f71d03f6-0047-447a-8a85-c279d86af2b5" />

У моделі виявлено два рекурсивні зв'язки:
–	Менеджер → Менеджер: атрибут supervisor_id (FK → manager_id). Відображає ієрархію підпорядкування (керівник → підлеглий менеджер). Необов'язковий (NULL для найвищого рівня);
–	Товар → Товар: атрибут parent_category (FK → product_id). Відображає ієрархію категорій товарів (категорія → підкатегорія). Необов'язковий (NULL для кореневих категорій).
